Basics

Compiling and Usage

To use nuMCMC, compile the program using the make command in the GNU terminal. Once compiled, the program can be used in any other program.

To use the MCMC sampling in another program, use the flag -lnu and refer to the location of the file using #include "(the path goes here)/nu.h".

Understanding the MCMC

The MCMC sampling algorithm aims to sample some probability distribution \(p(\vec{x})\), where \(\vec{x}\) is an N-dimensional vector. One advantage of using affine invariant MCMC is that you don't need to have prior knowledge of the distribution for all the variables. There are three main rules when running MCMC:

  • Choose your initial guess;
  • Ensure convergence;
  • Get as many samples as you need.

In the algorithm, we consider two elements: walkers and ensembles of walkers. A walker is a vector in the N-dimensional space that stores one sampling story. An ensemble is a set of walkers. In the beginning, you choose an ensemble of walkers with randomly organized initial positions.

As an example, let's consider the function

\(-\frac{1}{2}\log \left[p(\vec{x})\right]=\sum_i \frac{(x_i-c_i)^2}{\sigma^2_i}-\frac{(x_0-c_0)(x_1-c_1)}{\sigma_0\sigma_1}-\frac{(x_2-c_2)(x_1-c_1)}{\sigma_2\sigma_1},\)

where \(c_0=50\), \(c_0=1\), \(c_0=20\), \(\sigma_0=1\), \(\sigma_1=1\) and \(\sigma_2=20\).

Running

To obtain more samples, we can run the MCMC algorithm for more steps using the run method (or the run_parallel method for parallelized running). For example, in Figure 2, we ran the algorithm for 50 steps. It is important to note that the function passed to the run method should be \(-\frac{1}{2}\log(p) \) to sample the probability distribution. 
                                        
...
my_sample.run(loglike, nsteps);  /* The serial run */
my_sample.run_parallel(loglike, nsteps, num_of_ threads); /* The parallelized run */
char header[256] = "x,y,z,walker\n"; 
my_sample.save_chain_walker(file_sample_name,header);
...
The method 'save_chain_walker' saves the walkers' history as a CSV file where the last column is the walker identifier, given by an integer ranging from 1 to the number of walkers. Fig. 2 shows the output results. Below, we present an interactive walk for the x and y parameters.

In Figure 2, you can observe that the history of the walkers is still visible in the path from the initial guess to the exact distribution. In order to ensure convergence, we need to take additional steps. This is the next topic that we will cover.

Convergence

Convergence time series
A way to check the convergence of the walkers is to plot their history in each parameter of interest. Convergence happens when the walkers stabilize their walks in a given neighborhood. To do this, you can run the algorithm once to get a history of the walkers, and then plot their time series for each parameter. If the walkers have not converged, you can use their final configuration as the initial configuration for another run, and repeat this process until convergence is achieved. The last line of the code "save_chain_walker", saves the history of the walkers, so you can plot the figure we show before.

The real probability distribution

We used the following code to generate the data for this figure: 
                        
...
nu::Mcmc my_sample(nwalkers, ndim, init_pos); 
my_sample.run(loglike, bournout); 
...
my_sample.reset(); 
my_sample.run(loglike, nsteps);
...

...
The diference in the last code is that we use the "reset" method. The reset method erases all the history of the walkers. So, we lose all the history of the walkers until reach convergence. Given that we can run again and get some sampling points.

Save and Load

In the nuMCMC it is also possible to save and load states. If the convergence takes a long time, you can save your present configuration state in a file and load it in a later usage. The code below exemplifies how can it be done.

                        
...
/*If you want to load a state */ 
nu::Mcmc my_sample(nwalkers, ndim, file_name); 
...
my_sample.run(loglike, nsteps); 
/*If you want to save a state */ 
my_sample.save_state(file_name);
...
